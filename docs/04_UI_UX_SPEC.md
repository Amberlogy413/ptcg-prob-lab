# 04 — UI/UX 規格

> 目標體驗:**「概率的計量儀器」**。用戶第一眼要感到「這個數字是被量出來的,不是被猜出來的」。權威來自克制:細髮絲線、表格數字、單一強調色、處處可驗算。本文 token 與規則為硬性;像素級細節可在不破壞方向的前提下發揮。

## §1 設計方向與反模式

方向關鍵詞:計量證書 × 實驗室紀錄 × 牌桌上的清醒。淺色「實驗紙」主題為預設(數學閱讀性最佳),深色主題列 V2。

**明令禁止(AI 千篇一律外觀)**:紫藍漸層 hero、玻璃擬態卡片海、無意義 emoji 點綴、每張卡都掛陰影、置中大標語+三欄 feature 卡的官網式排版、彩虹圖表。陰影幾乎不用——以髮絲線(hairline)分隔;**整個產品只有「數學收據」允許一層紙質陰影**,讓簽名元素自己發光。

## §2 設計 Token(寫入 `tailwind.config`)

```js
colors: {
  paper:   "#FAF8F3",  // 全域背景(暖紙)
  surface: "#FFFFFF",  // 卡片
  ink:     "#15181C",  // 主文字
  ink2:    "#5A6069",  // 次文字
  line:    "#E3DFD6",  // 髮絲線
  blue:    "#2B59C3",  // 唯一強調色「製圖藍」:主按鈕、作用中 chip、連結
  good:    "#0E7A4A",  // 翡翠:有利/上升 delta/「理想手」
  warn:    "#B45309",  // 琥珀:注意/「可用手」
  bad:     "#B3261E",  // 緋紅:不利/下降 delta/「死手」
  receipt: "#FFFEF9",  // 收據紙
}
```

語義色只用於「概率好壞與差異」,絕不作裝飾。對比度:全部文字組合 ≥ 4.5:1(ink/ink2 於 paper、白字於 blue 均已達標)。

字體:
- 介面與中文:`"Noto Sans TC", "IBM Plex Sans", system-ui`。
- **所有數字、分數、收據**:`"IBM Plex Mono"`,並開 `font-variant-numeric: tabular-nums`——數字對齊是本產品的人格。
- 字級階:12 / 13.5 / 15(本文)/ 18 / 24 / 36 /(頭條數字)`clamp(40px, 9vw, 72px)`。
- 間距:4px 階(4/8/12/16/24/32/48);圓角:控制元件 6px、卡片 10px;邊框 1px `line`。
- 動效:80–160ms ease-out,僅限展開/收合與數值滾動;尊重 `prefers-reduced-motion`。

## §3 資訊架構

頂列工作區:**牌組 ▸ 提問 ▸ 獎賞卡 ▸ 比較**(V2 加「追蹤器」)。左欄常駐「牌組摘要」:總數 60 驗證、基礎寶可夢數、重抽率小錶(B=10 → 25.86%)。右主區 = 查詢建構器 + 結果串。頁腳:粉絲工具與 IP 聲明(PRD §7)+「精確計算 · 非模擬」徽章 + 黃金測試數。

## §4 句子式查詢建構器(核心互動)

把查詢拼成一句人話,每個【】是控件(chip/下拉/數字步進):

> 在【起手 7 張】中,我想知道【火球鼠 ×4】出現【至少 1 張】**且**【超夢風暴 ×3】出現【至少 1 張】的概率,【✓ 含重抽修正】。

規則:
- 卡片 chip 從牌組挑選(顯示卡名與投數);「＋且」最多加到 5 張卡。
- 約束下拉:至少 n / 恰好 n / 至多 n / 介於 a–b / 避開(0 張);n 步進器即時夾限於 [0, min(投數,7)]。
- 「含重抽修正」是醒目開關 chip:開=藍底白字「已含重抽修正」,關=描邊「未含重抽修正」;切換即重算並以 ±pp 徽章顯示兩值差。追蹤卡未標記是否基礎時,開關置灰並引導去標記。
- 獎賞卡工作區同構句型:「在【6 張獎賞卡】中,【…×4|手上已有 1 張】被放入【至少 1 張】的概率」,模式三選一(賽前無條件/已知手牌/賽前·含重抽)以分段控件呈現,旁附 ⓘ 一句話解釋。
- 預設十問:builder 上方一排快捷 chip,一鍵填入(PRD §4-14)。

## §5 結果卡(三層,由上而下)

1. **頭條層**:左為巨型 mono 數字(如 `15.383618%`),下方堆疊另兩格式 `11011691/71580630 · 1 in 6.50`;右為**精密量尺**——一條帶 0–100% 刻度(每 10% 主刻、5% 次刻)的水平尺,藍色游標壓在數值上。一句白話判讀(i18n):「真實對局中,約每 6.5 場有 1 場開局即達成。」若為條件化結果,頭條下方掛灰色對照行:「未含重抽:11.404965%(−3.98pp)」。
2. **分布層**:完整聯合表 + 圖。表欄:`組合(k_A,k_B)` | `概率` | `精確分數` | `1 in N` | 行內微長條;滿足約束的行以左側 3px 藍邊標記;預設按概率降序,可切換字典序;>30 行折疊為「展開全部 n 行」。圖:長條(分布)或階梯曲線(回合),自製 SVG、單色藍、hover 顯示精確值。
3. **收據層(簽名元素)**:「展開數學收據 ▾」。收據用 `receipt` 紙色、mono 字體、頂部虛線撕裂邊、唯一允許的紙陰影。內容五行式:
   ```
   公式      P = Σ C(4,a)·C(3,b)·C(53,7−a−b) / C(60,7)  (a≥1, b≥1)
   代入      C(60,7) = 386,206,920
   合計      分子 44,046,764 → 既約 11011691/96551730
   條件化    ÷ p_valid 216911/292581 = 11011691/71580630
   檢核      ✓ 與黃金向量 combo_valid_A4basic_B3_ob6 一致
   ```
   底部小字:「全程 BigInt 精確分數;小數為 round-half-up 顯示。」收據可一鍵複製為純文字。

## §6 其他關鍵元件

- **重抽儀表板(Q1)**:三聯小卡(重抽率/期望次數/恰 m 次的迷你幾何分布),搭配條件分布表與「你實際拿到的手」標題——教育點:k=0 行永遠是 0。
- **牌組編輯器**:行 = 投數步進 + 卡名 + 「基礎」切換 + 刪除;頂部 60 計數環;匯入精靈兩步(貼上 → 標記基礎)。
- **比較視圖**:左右兩欄同一查詢,中央 delta 欄(`+3.98pp` 綠/紅);支援「複製牌組 A 並改一行」快速分叉。
- **敏感度滑桿**:選一張卡,投數 0–4 掃描,迷你折線 + 表;標題自動生成「第 4 投值多少?」。
- **蒙地卡羅驗證(V2)**:結果卡角落「驗證 ▸」,Worker 跑 100,000 局,顯示模擬值、95% 誤差帶與精確值的重合動畫;文案:「模擬會抖動,精確值不會。」
- **追蹤器(V2)**:牌組清單逐張「已見」勾選,右側即時後驗表;頂部常駐賽事合規提醒(PRD §4-16)。

## §7 微文案(zh-Hant 基準;全部進 i18n)

| 鍵 | 文案 |
|---|---|
| badge.exact | 精確計算 · 非模擬 |
| toggle.mulligan.on / off | 已含重抽修正 / 未含重抽修正 |
| hint.mulligan.naive | 未含重抽:{value}({delta}) |
| empty.deck | 還沒有牌組。貼上 PTCG Live 牌表,或從第一張卡開始。 |
| error.deckCount | 目前 {n}/60 張——仍可計算,但結果對應的是 {n} 張牌庫。 |
| error.basicUnknown | 請先標記哪些是基礎寶可夢,重抽修正才能啟用。 |
| receipt.footer | 全程 BigInt 精確分數;小數為 round-half-up 顯示。 |
| tracker.legal | 正式比賽使用外部工具可能違反賽事規定;本功能供練習與覆盤。 |
| q1.conditional.title | 你實際拿到的手(已含重抽) |

語氣準則:像可靠的裁判長——短句、肯定、不賣萌、不驚嘆號;解釋永遠跟著數字。

## §8 無障礙與響應式

- 全鍵盤可達;焦點環 2px 藍;表格有 `<caption>` 與 `scope`;量尺與圖表附文字等價(aria-label 含精確值)。
- 觸控目標 ≥44px;<768px 時 builder 轉為逐步精靈(選卡 → 選約束 → 開關修正 → 結果),左欄收進抽屜;頭條數字縮至 40px 仍為畫面主角。
- 列印樣式:結果卡+收據單頁輸出,隱藏導覽。
