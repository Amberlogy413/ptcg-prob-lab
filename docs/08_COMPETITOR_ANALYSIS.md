# 08 — 競品深度分析:ptcgtw.shop(奶爸工作室)

> 目的:依產品擁有者指示(2026-06-12)「深度學習市場上最成功的案例,先做到佢所有功能,
> 然後再開發革命性功能來超越它」。本文件是研究結論與策略裁決;對齊與超越的工程排程
> 見 `06_ROADMAP.md` Phase 8–9。
>
> 研究方法:29 個研究代理對該站 24 個頁面/端點逐頁抽取功能(含直接讀取原始 HTML 與
> 內嵌 JavaScript),另沿四條軸線掃描社群口碑(巴哈姆特、PTT、Threads、Facebook、
> YouTube)與整個繁中/英/日工具生態。所有關鍵結論均有來源;無法證實之處明確標示。

## 1. 對象速寫

**ptcgtw.shop「繁中PTCG卡表生成器-奶爸工作室」**是台灣(兼及香港)繁中 PTCG 社群
事實上的標準工具箱。單人(站長自稱「奶爸」)業餘維護的 PHP + MySQL 網站,免費、
免帳號,以廣告(ClickForce)、蝦皮周邊、OPay/Vocus 捐款、KIWIPTCG 卡店導購
(專屬折扣碼、追蹤式轉址)變現。社群滲透極深:PTT 新手帖直接推薦「先在線上組牌、
測牌」;香港 Threads 廣東話入坑指南列為必用;有第三方教學片、甚至 memes.tw 梗圖
模板。官方 Facebook 專頁僅約 2,388 讚——其地位來自口碑與 SEO,非社群經營。

成功核心:**解決官方真空**。官方訓練家網站當年只有不可搜尋的 PDF 卡表;奶爸補上
繁中卡片資料庫,再沿「搜卡 → 組牌 → 估價 → 印測試牌 → 測起手 → 對戰網 → 交構築表」
整條本地玩家工作流逐站築壩,並以「存到官方網站」一鍵整合(自動降稀有度版本、回傳
官方牌組代碼)鎖住競技玩家。

## 2. 完整功能清單(逐頁實證)

### 2.1 卡片資料庫與搜尋(首頁,核心)
- 四路全文搜尋:卡名/特性/招式/卡片文字(空格分隔多關鍵詞)。
- 篩選:系列(至 2026 年最新 M6/J 標,含 PR、口袋版、玩家手繪)、類別、屬性、
  進化階段、特殊規則(古代/未來/一擊/連擊…),進階十項(撤退費、弱抵、招式費用、
  HP 範圍、傷害、特性有無、稀有度 18 級、高版本)。
- 單卡視窗:高解析卡圖下載、中日英卡名複製、進化鏈搜尋、判例連結、蝦皮/日幣參考價、
  KIWI 購買鍵、單卡分享連結。
- 資料含中/日/英三語卡名與雙市場價格;判例(官方裁定)隨新系列更新。

### 2.2 牌組建構與輸出
- 加卡/移卡/張數/排序、按類別自動分組、60 張計數。
- 伺服器端存檔短碼分享(`?s=AB1234`,免登入)。
- 讀檔三格式自動偵測:奶爸短碼、台灣官方 20 碼、日本官方碼(直連兩地官網 API)。
- 「存到官方網站」:一鍵轉存官方 deck-build,回傳官方代碼與官方牌圖。
- PTCG Live 雙向轉換(中→英匯出、英→中匯入)。
- 高/低稀有度版本一鍵互換(連動價格)。
- 輸出:整副牌一張圖(三種版式)、紙本比賽構築表(套印選手資料,ibon 列印)、
  測試用代牌 PDF、Tabletop Simulator 牌組檔。
- 「弔念G標」輪替預覽:灰掉即將退環境的卡。

### 2.3 「測起手」(佢唯一的概率相關功能——已讀原始碼證實)
- 模組 `07_03_modal_start_hand`:`Math.random` Fisher–Yates **單次**洗牌,
  顯示起手 7 張 + 第一抽 1 張 + 獎勵牌 6 張(卡圖)。
- **無任何統計、無重抽(Mulligan)檢查、無概率輸出**;唯一驗證是「夠唔夠 60 張」。
- 官方 QA 原文:「想知道關鍵牌卡獎的機率嗎?不用再算了,按下測起手,按個100次就知道
  機率囉」(引文照錄)——即著用戶以手按百次當蒙地卡羅。15% 級事件按 100 次的
  95% 信賴帶約 ±7 個百分點;且彼站用語為「機率」「獎勵牌」,非本品術語鎖。
- **此即本產品的整個存在理由所在**:對方自己驗證了「玩家想知道起手與卡獎概率」
  這需求,但只給了一台抽樣玩具。

### 2.4 對戰網(battle_v4)
- 免帳號「登入」:一條新手測驗題(防亂入)+ 暱稱 + 牌組短碼;localStorage 記憶。
- 人手沙盒:自由移卡、無規則引擎(巴哈實錄「網頁本來就沒有特別去寫防呆」)、
  棄手牌等效果靠雙方配合操作;配對靠 LINE/Discord 群,無站內配對。
- v3 與 v4 並存;頁面掛五個廣告位。

### 2.5 內容與社群資產
- 日本賽果牌組庫:約 2 萬副(2,007 頁),日刊更新,圖片熱鏈自 pokekameshi.com,
  卡名譯成繁中做成可搜尋標籤;PJCS 等大賽入賞整理。
- 推薦牌組(站長精選,開放 API 63 頁)、全台店家/賽事目錄(約 160 店,人手維護)、
  YouTuber 推薦表(Google 表單收集)、判例搜尋、罐頭 QA 複製板、
  卡價追蹤 Telegram(變動 >3% 且 >NT$10 推播)、系列卡價表 + 開盒價值分析。

## 3. 社群口碑摘要

**讚**:一站式生態黏性極高;「存到官網」是殺手鐧;免費免登入(資料庫部分);
繁中資料維護勤(判例隨 M5 更新);台港兩地新手指南都推薦。

**彈**:對戰網要登入——競品 ptcg-tw-sim.com 直接以「免安裝、免登入、即開即戰」做
賣點打此摩擦;無規則引擎、新手「搞了個亂七八糟」、求圖文教學而不得;文檔薄弱
(教學靠 2022 年外部 YouTube 片);**IP 灰色地帶**——巴哈討論明言「奶爸本身就已經
有下廣告營利,只是沒有被抄而已」,創作者被勸喻拍片改用官方 Live;單人維護
bus factor;Dcard 年輕族群零滲透。

## 4. 生態掃描:概率工具是全球性真空

沿「邊個做緊抽卡/卡獎數學」掃了三語生態,結論:

| 工具 | 語言 | 數學能力 | 缺口 |
|---|---|---|---|
| ptcgtw.shop 測起手 | 繁中 | 單次洗牌視覺化 | 無任何計算 |
| 官方 deck-build(+民間 Tampermonkey 補丁) | 繁中 | 補丁僅隨機發牌 | 無任何計算 |
| Limitless DrawCalc | 英(無中文) | 超幾何,5 條固定題 | 無組合題、無卡獎、浮點 |
| yazirusis 確率計算機 | 日 | 多卡 AND/OR、回合曲線 | 重抽模型係其他遊戲的「可選換牌」,**非 PTCG 強制重抽**;捨入百分比 |
| zk-phi / drawlator / pokecalculate / deckoptimize | 日/英 | 各式超幾何 | 場景窄、浮點、無中文 |

**冇任何一個工具——任何語言——做緊:**(1) 重抽條件化的聯合概率(全部停在
「單獨一條重抽率」或索性無視,系統性低估真實賠率,即 02 §4 的 11.40% vs 15.38%);
(2) 已知手牌條件化的獎賞卡期望;(3) 精確分數/「1 in N」/數學收據。本產品所修正的
謬誤清單(02 §9)在每一個現存工具入面都係現在進行式。

## 5. 鐵律過濾:佢的功能逐項裁決

「做到佢所有功能」必須過三道閘:全客戶端(CLAUDE.md #7)、IP 安全(#6)、精確數學
靈魂(#1)。逐項分類如下(裁決理由記入 `DECISIONS.md` 2026-06-12):

### A. 對齊——可以做,而且要做得比佢好(→ Phase 8)
| 佢的功能 | 我們的合規版 | 超越點 |
|---|---|---|
| 測起手(單次洗牌) | **試抽桌**:種子化誠實發牌(7+1+6),文字卡片,真重抽循環 | 每一手附**精確概率註解**;批次 ×N 統計疊精確值與信賴帶——把「按 100 次」變成「睇住樣本收斂到精確值」的教學 |
| PTCG Live 匯入 | 已有;再加容錯格式(「4 卡名」「卡名 x4」純文字行) | 內建「由奶爸工作室搬牌組過嚟」三步指南(佢站「轉換成Live英文」→ 貼入本品)——做佢生態的**審計層**,唔搶佢的構築層 |
| 推薦牌組庫 | **範本牌組庫**:隨 app 內建的純文字典型構築(卡名+投數+基礎標記),一鍵載入 | 載入即出全套精確體檢,而非一張圖 |
| 牌組一張圖輸出 | 文字版牌表圖卡(本品設計 token,Canvas) | 圖卡自帶重抽率/分級等精確徽章 |
| 紙本構築表 | 文字版賽事登錄牌表(原創排版,列印樣式 + 套印選手欄位) | 離線可用;不抄官方表格版面 |
| 弔念G標 | 牌組行可選「賽制標記」欄,一鍵預覽輪替後牌組 + **重算輪替後全套概率** | 佢只灰掉卡;我們答「輪替後你副牌數學上爛咗幾多」 |
| 多語 | zh-Hant/en 已有 | — |

### B. 明確唔做(連同點解)
- **卡圖資料庫/卡圖下載/代牌列印**:IP 鐵律。佢的最大護城河正正是佢的最大法律風險
  (社群自己都咁講);本品以文字卡名身份立足,係防禦性資產,不是缺陷。
- **卡價/開盒 EV/Telegram 推播/高低版本換算**:需要爬蟲、價格資料庫、伺服器排程,
  全部撞零後端鐵律;且係市場情報,不是對局數學。
- **對戰網(多人模擬)**:即時連線必須後端;人手沙盒無規則引擎亦偏離「精確儀器」
  定位。單人「測牌」需求由試抽桌(多回合 goldfish 模式,Phase 9)覆蓋。
- **官方網站 API 整合(存到官網/讀官方碼)**:跨域 + 後端代理需求;以 PTCG Live
  文字格式做橋(官方碼→佢站/Live 客戶端→文字→本品)。
- **判例庫、店家目錄、YouTuber 表、社群群組**:內容營運,非工具;與精確數學定位
  無關。互補不重做。

### C. 革命性超越(→ Phase 9;全部係佢冇、而且以佢架構好難有)
1. **牌組數學體檢報告**:一鍵全套——重抽率、起手分級、關鍵組合(重抽條件化)、
   能量斷流、死手歸因 Top 3、獎賞風險——一頁可分享圖卡。「60 秒體檢你副牌。」
   全部組件已存在(Phase 2–7),純組合 + UI。
2. **試抽桌進階:逐回合 goldfish + 劇本檢查**:逐回合抽牌,旁邊實時疊「到第 T 回合
   見到 X 的精確曲線」(02 §6)與接力事件(02 §6.5)——模擬與精確並排,直覺與
   真值互相校準。
3. **訓練閉環**(已有 B1/B2/C1 資產):試抽桌出題 → 心算 → 精確揭曉,將佢「按
   100 次」的用戶直接升維。

## 6. 定位結論

唔同佢打正面。奶爸贏在**資料庫 + 生態 + 本地化服務**,呢啲要後端、要 IP 冒險、要
日日營運——三樣本品都唔做。本品贏在**精確數學層**,呢樣佢(同全世界)技術形態上
做唔到(PHP 字串拼 SQL 的架構唔會有 BigInt 有理數核心 + 雙實作黃金驗證)。

口號級定位:**「佢度砌牌,嚟我度知賠率。」**(奶爸 = 構築層,本品 = 審計層)
匯入橋做到零摩擦,佢的兩萬副日本牌組庫反而變成本品的上游流量。

風險誠實註記:(1) 奶爸一個人都維護到判例日更,佢加一個「概率計算」選單項的成本
唔高——但冇黃金管線同精確核心,佢只能加浮點近似版;本品的可驗證性(收據、分數、
雙實作)係抄唔到的差異。(2) 本品冇卡圖,視覺吸引力天然蝕底;體檢報告圖卡的設計
質素(04 §)就係補償手段。(3) 所有「佢站引文」均為研究紀錄;UI 不得出現對方品牌。
