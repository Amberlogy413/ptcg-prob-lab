## §0 記號與模型假設

- N = 60(牌組張數)、H = 7(起手張數)、P = 6(獎賞卡張數)。除非特別說明,皆用此預設;所有函式以參數形式保留 N/H/P 以利測試。
- C(n,k) 為二項式係數;約定 k<0 或 k>n 時 C(n,k)=0。
- 洗牌假設:**完美均勻洗牌**——60 張牌的所有排列等概率。由此得「**可交換性**」:任取牌序中固定位置的任意 j 張,其組成是 60 張中均勻隨機的 j 子集,與位置標籤無關。本文多處依賴此引理(§5.1、§5.5、§6.2)。
- 對局流程(現行官方規則,2026-06 查證):抽起手 7 張 → 若 7 張中**沒有任何基礎寶可夢**則為 Mulligan(重抽):亮牌、洗回、重抽 7 張,重複直到出現 ≥1 張基礎寶可夢(每次重抽,對手可多抽 1 張作為補償)→ 從牌庫頂取 6 張面朝下作為獎賞卡 → 開始對局。每位玩家(含先攻玩家第一回合)在自己回合開始時抽 1 張;先攻玩家第一回合的限制是「不能攻擊、不能使用支援者卡」,**不是**「不抽卡」。
- 全部概率以 BigInt 既約分數表示與運算;浮點僅允許出現在顯示層(§7)。

## §1 單類別超幾何分布

牌組中某卡有 x 投,從 N 張抽 n 張:

```
P(恰好 k 張) = C(x,k) · C(N−x, n−k) / C(N,n)
P(至少 w 張) = 1 − Σ_{k=0}^{w−1} P(恰好 k 張)
```

實作:`hypergeomPmf / hypergeomAtMost / hypergeomAtLeast`(`src/lib/prob/hypergeom.ts`)。

**錨點**:4 投卡在起手 7 張中至少 1 張 = 1 − C(56,7)/C(60,7) = **38962/97527 ≈ 39.949963%**。(社群口耳相傳的「4 投約四成」,本工具給出精確值。)

## §2 多類別超幾何(聯合分布)

追蹤 m 種卡,投數 c₁…c_m,其餘 r = N − Σcᵢ 張為「其他」類。抽 n 張中各出現 k₁…k_m 張:

```
P(k₁,…,k_m) = [ Π_i C(cᵢ,kᵢ) ] · C(r, n − Σkᵢ) / C(N,n)
```

實作:`multiHypergeomPmf`;組合枚舉用 `compositions(maxes, total)` 生成所有 0 ≤ kᵢ ≤ min(cᵢ,n) 且 Σkᵢ ≤ n 的向量。複雜度:表格大小 ≤ Π(min(cᵢ,7)+1);m ≤ 5、cᵢ ≤ 4 時上界 5⁵ = 3125 格,實際因 Σ ≤ 7 遠小於此,毫秒級完成(預算見 `03 §6`)。

## §3 Q1 — 基礎寶可夢與 Mulligan(重抽)

牌組含 B 張基礎寶可夢。

- 原始分布:`dist[k] = C(B,k)·C(60−B,7−k)/C(60,7)`,k = 0…7。
- 重抽概率 `q = dist[0] = C(60−B,7)/C(60,7)`;有效起手概率 `valid = 1 − q`。
- **條件分布(你實際拿到的手)**:`P(k | 有效) = dist[k]/(1−q)`(k ≥ 1),k = 0 時為 0。
- 重抽次數 M 服從幾何分布:`P(M = m) = q^m·(1−q)`;**E[M] = q/(1−q)**。
  推導:E[M] = Σ_{m≥1} m·q^m(1−q) = q/(1−q)(標準幾何級數)。
- 期望基礎張數(未條件化):由線性 E[K] = 7B/60(每張牌位含基礎的概率為 B/60)。

實作:`openingBasics`、`mulliganCountPmf`(`opening.ts`)。

## §4 Q2 — 追蹤卡組合(含 Mulligan 條件化)

### §4.1 約束語義
每種追蹤卡 i 給定 [minᵢ, maxᵢ](0 ≤ minᵢ ≤ maxᵢ ≤ 7)。事件 E =「對所有 i,minᵢ ≤ kᵢ ≤ maxᵢ」。「至少 1」= [1,7];「恰好 2」= [2,2];「避開」= [0,0]。

未條件化:`P(E) = Σ_{k ∈ E} P(k₁,…,k_m)`(§2 公式)。同時輸出**完整聯合表**(所有組合、各自概率、是否滿足),這是「詳列各可能性」需求的實現。

**錨點**:A4+B4 各至少 1 = **1403917/9655173 ≈ 14.540568%**;A4+B3+C2 各至少 1 = **7625861/386206920 ≈ 1.974553%**;4 投恰好 2 張 = 9646/162545 ≈ 5.934357%。

### §4.2 Mulligan-aware(條件於有效起手)
真實對局只會給你「≥1 張基礎」的手。把牌組分為:追蹤卡(各 cᵢ,並標記是否基礎)、**其餘基礎** ob 張、其餘非基礎 rest = N − Σcᵢ − ob 張。對 (k₁,…,k_m, j)(j = 抽到的其餘基礎張數)枚舉:

```
手牌有效 ⟺ Σ_{i: 基礎} kᵢ + j ≥ 1
p_valid = Σ_{有效組合} P(k, j)
P(E | 有效) = P(E ∧ 有效) / p_valid
```

實作:`comboOpening(cards, { mulliganAware: { otherBasics } })`,回傳 `event`(條件化)、`pValid`、`eventUnconditioned`(對照)與條件化後的完整表。

**殺手錨點**:A(4 投,基礎)+ B(3 投,非基礎),其餘基礎 6 張(全組 10 基礎),「各至少 1」:
- 天真(忽略重抽):11011691/96551730 ≈ **11.404965%**
- 真實(條件於有效起手):11011691/71580630 ≈ **15.383618%**(p_valid = 216911/292581 ≈ 74.137077%)

注意分子相同並非巧合:此事件要求 A ≥ 1,而 A 是基礎,故事件 ⊆ 有效,於是 P(E|有效) = P(E)/p_valid。一般情形(事件不蘊含有效)必須走完整的聯合枚舉,不可套用此捷徑——實作已正確處理一般情形,本例只是漂亮的可驗算特例。**產品啟示:忽略重抽的計算器在這類查詢上系統性低估近 4 個百分點;這正是本工具存在的理由,UI 應並排展示兩值。**

## §5 Q3 — 獎賞卡

### §5.1 模式 (a):無條件(賽前)
**可交換性引理**:獎賞卡是「發完手牌後牌庫頂 6 張」,但對外部觀察者(尚未看任何牌)而言,牌序均勻 ⇒ 任何固定 6 個位置的牌是 60 張中均勻隨機的 6 子集。故:

```
P(x 投卡恰有 k 張被獎賞) = C(x,k)·C(60−x,6−k)/C(60,6)
E[被獎賞張數] = 6x/60 = x/10
```

證明梗概(離散驗證亦寫入黃金產生器的 assert):先發手牌再取獎賞、對手牌邊際化,結果與直接套上式逐項相等。

**錨點**:x=1 → P(被獎賞) = **1/10 = 10% 整**(「你唯一的 ACE SPEC 有正好一成概率躺在獎賞卡裡」);x=2 → 113/590 ≈ 19.152542%;x=3 → 2354/8555 ≈ 27.516072%;x=4 → 57128/162545 ≈ **35.145960%**(1 in 2.845)。

### §5.2 模式 (b):已知手牌
手上已見 h 張,剩 x−h 張在未見的 M = 53 張中;獎賞為其中 6 張:

```
P(恰 k 張被獎賞 | 手上 h 張) = C(x−h,k)·C(53−(x−h),6−k)/C(53,6)
```

**錨點**:x=4, h=1 → ≥1 被獎賞 = 7211/23426 ≈ **30.782037%**;x=4, h=0 → 22892/58565 ≈ **39.088193%**(「七張裡一張都沒看到」反而推高了被獎賞的懷疑——後驗直覺的第一課)。

### §5.3 模式 (c):賽前 Mulligan-aware
對「最終保留的手有效」條件化、對手牌內容邊際化。以 §4.2 的分解(此處追蹤單卡 x,標記是否基礎,其餘基礎 ob):

```
P(恰 k 被獎賞 | 有效) = Σ_{有效手 (h,j)} P(h,j) · C(x−h,k)·C(53−(x−h),6−k)/C(53,6) ÷ p_valid
```

實作:`prizeDistPreGame(x, { isBasic, otherBasics, conditionOnValid: true })`。

**方向相反的雙錨點**(證明此修正不可省):
- x=4 **基礎**卡、其餘基礎 6:E[被獎賞|有效] = 21933186/57481415 ≈ **0.381570** < 0.4;P(≥1) ≈ 33.770701%。直覺:有效手偏向「多抓了基礎」,留給獎賞的基礎變少。
- x=4 **非基礎**卡、其餘基礎 10:E ≈ **0.403686** > 0.4。直覺:有效手至少佔走 1 個基礎牌位,非基礎卡留在牌庫的比例略升,被獎賞概率隨之略升。

### §5.4 多卡聯合(已知手牌)
各追蹤卡剩餘 cᵢ − hᵢ 張在 53 張中,對獎賞 6 張套 §2 多類別公式,支援各卡 [min,max] 被獎賞約束與完整表。實作:`prizeJointGivenHand`。
**錨點**:A4 手 1、B3 手 0,「各至少 1 被獎賞」= 273379/3279640 ≈ **8.335640%**;「皆未被獎賞」(cell 0,0)= 1533939/3279640。

### §5.5 對局獎賞卡追蹤器(V2)的後驗原理
對局任意時點,設你已確認見過自己牌組的某些牌(起手、每回合抽牌、檢索/翻牌效果所揭示…)。未見集合 U(大小 u)= 牌庫未見部分 ∪ 獎賞卡。再用可交換性:**給定已見資訊,獎賞卡是 U 中均勻隨機的 6 子集**(你的抽牌順序不會洩漏「哪幾張」被劃為獎賞)。故某卡未見 u_x 張時:

```
P(恰 k 張被獎賞) = C(u_x,k)·C(u−u_x,6−k)/C(u,6),  E = 6·u_x/u
```

注意:對手的行為不提供關於**你的**牌序資訊;但你自己的效果(如看牌庫頂 X 張後洗回)需按「已見但已洗回」正確處理——洗回後該資訊失效,追蹤器只應記「當前確定不在 U 中」的牌。UI 必附賽事合規聲明(PRD §4-16)。

## §6 回合曲線

### §6.1 已見張數
到「我的第 T 回合抽牌後」,自然已見張數:

```
nSeen(T) = 7 + T                      (現行規則,先攻後攻皆同)
nSeen(T) = 7 + T − 1   (先攻且啟用歷史規則 firstPlayerSkipsFirstDraw)
```

規則查證(2026-06):現行規則先攻第一回合**有**抽卡;「先攻第一回合不抽」屬 2004–鑽石珍珠時代的歷史規則,以選項保留、預設關閉。另支援 `extraSeen` 偏移(用戶自報的額外抽牌)。物理上限:一局最多見 60 − 6 = 54 張。

### §6.2 概率
由可交換性,你依序見到的前 n 張是 60 張中均勻 n 子集(獎賞卡被劃走不影響前 n ≤ 54 張的邊際分布):

```
P(前 n 張中 ≥ w 張 X) = hypergeomAtLeast(60, x, n, w)
```

實作:`cardsSeenByTurn / pSeenAtLeast / seenCurve`(`turns.ts`)。

**錨點(x=4,≥1 張)**:n=7 → 39.949963%;n=8 → 44.482041%;n=9 → 48.752653%;n=10 → 52.772053%;n=11 → 56.550289%;n=12 → 60.097204%;n=13 → 63.422437%。(後攻第 6 回合 n=13:「六回合內見到 4 投卡」約 63.4%。)

### §6.3 誠實聲明
本曲線**未**對 Mulligan 條件化(條件化會微幅改變前 7 張的組成,對含基礎卡的查詢有二階影響)。UI 須標示「未含重抽修正」;Mulligan-aware 回合曲線列為待辦(對有效手分布加權後接力枚舉),屬 V2 之後的擴充,加入時必須先擴充黃金產生器。

## §7 有理數表示與顯示捨入

- `Rat = { n: bigint, d: bigint }`,不變量:d > 0、gcd(|n|,d)=1(`rational.ts` 建構即化簡)。
- 顯示三格式:百分比、精確分數 `n/d`、「1 in N」(N = d/n 捨入)。
- 小數一律 **round-half-up(逢五進位、遠離零)**,於 BigInt 內完成:q = ⌊|n|·10^places / d⌋,餘數 ×2 ≥ d 則進位(`format.decimalStr`)。黃金向量的 `*_dec` 欄位即此規則 15 位,TS 輸出需逐字元相等(已驗證)。
- 圖表允許 float,唯一橋接點 `toChartNumber`(經 15 位精確字串轉換)。

## §9 常見謬誤(實作與文案都要避開)

1. **獨立相乘謬誤**:P(A≥1 且 B≥1) ≠ P(A≥1)·P(B≥1)。同手牌內不同卡張數**負相關**(名額互搶):4+4 案例,相乘給 39.949963%² ≈ 15.959995%,精確值 14.540568%——高估 1.4pp。
2. **有放回(二項)近似**:1 − (56/60)⁷ ≈ 38.303939%,精確值 39.949963%——低估 1.6pp。60 張的小牌庫,超幾何與二項差距不可忽略。
3. **忽略 Mulligan**:§4.2 已示範低估 3.98pp。凡涉及起手的問題,預設都該條件化。
4. **「重抽不影響獎賞卡」**:§5.3 證偽,且方向視卡是否基礎而反轉。
5. **「每張獎賞卡 1/10 所以 6 張就是 60%」**:期望可加(E = x/10 正確),概率不可這樣加;x=4 的 P(≥1 被獎賞) 是 35.15%,不是 40%。
6. **先攻不抽卡**:歷史規則,現行規則先攻第一回合會抽。曲線預設用現行規則。
7. **顯示捨入回灌計算**:任何後續計算必須從精確分數出發,嚴禁拿顯示字串再算。
8. **「模擬 10 萬次就是答案」**:蒙地卡羅只用於驗證與教學(`05 §D`),產品答案永遠是精確值。
