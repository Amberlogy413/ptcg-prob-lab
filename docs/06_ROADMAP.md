# 06 — 開發路線圖(ROADMAP)

> 每個 Phase 都有「完成定義(DoD)」;DoD 不滿足不得進入下一 Phase。每 Phase 收尾執行 `/ship-check` 與 `05 §F` 手動清單。**全程鐵律:黃金測試恆綠。**

## Phase 0 — 腳手架與核心搬遷

目標:把種子變成可開發的 Vite 專案,數學核心原封不動接上測試管線。

任務:
1. `npm create vite@latest`(react-ts)就地初始化;裝 Tailwind、Zustand、Vitest、Testing Library、ESLint(flat)+ Prettier。
2. `tsconfig` 依 `03 §4` 設定(`allowImportingTsExtensions` 等);**`src/lib/prob/` 一字不改**。
3. 把 `scripts/verify_seed.ts` 的斷言邏輯移植為 `tests/golden.spec.ts`(讀同一份 JSON);Node 直跑版保留。
4. 建 `package.json` scripts:`dev / build / test / verify:seed / golden / lint / typecheck`。
5. Tailwind token 依 `04 §2` 落地;載入 Noto Sans TC / IBM Plex Sans / IBM Plex Mono(本地字型檔,離線可用)。

DoD:`npm run typecheck && npm test && npm run verify:seed` 全綠;`npm run dev` 顯示帶 token 的空殼頁(頂列四工作區、頁腳聲明與「精確計算 · 非模擬」徽章)。

## Phase 1 — 牌組編輯器與匯入

目標:60 張牌組的建立、保存、匯入匯出與基礎寶可夢標記(後續一切數學的輸入)。

DoD:貼入真實 PTCG Live 牌表 → 標記基礎 → 左欄摘要即時顯示基礎數與重抽率小錶(調用 `openingBasics`);重載與離線後資料還在;匯入夾具測試通過(`05 §E`)。

## Phase 8 — 對齊期:吸收市場標準工具箱的可合規功能(`08 §5A`)

> 依據 `08_COMPETITOR_ANALYSIS.md` 的鐵律過濾結果,對齊 ptcgtw.shop 中通過
> 三道閘(全客戶端/IP 安全/精確數學)的功能,且每項都要附上對方做不到的精確
> 數學註解。不可對齊項及理由見 `08 §5B`,不重複排程。

任務:
1. **P8.1 試抽桌(TrialView)**:種子化誠實發牌——起手 7 張 + 第一抽 + 獎賞卡
   6 張,文字卡片(基礎寶可夢高亮);真重抽循環(零基礎即重發並計數,牌組無基礎
   時明確告知不進入循環);×1/×10/×100 批次模式,樣本統計疊**精確值與 95% 信賴帶**
   (調用 `openingBasics`/現有 MC 引擎);種子可見、可輸入、可重現。
2. **P8.2 匯入橋 + 範本牌組庫**:匯入精靈容錯擴充(「4 卡名」「卡名 x4」純文字行);
   內建「由外部工具搬牌組」三步指南;隨 app 內建純文字範本牌組庫(典型構築骨架,
   卡名+投數+基礎標記),一鍵載入即出摘要。
3. **P8.3 牌表輸出**:文字版牌組圖卡(Canvas、設計 token、自帶重抽率/分級徽章)+
   賽事登錄牌表列印頁(**原創排版**、選手資料套印欄、列印樣式;全程無卡圖)。
4. **P8.4 輪替預覽**:牌組行可選賽制標記欄;預覽模式灰掉退環境卡;一鍵分叉出
   「輪替後牌組」並重算全套概率(對照徽章顯示重抽率/分級變化)。

DoD:試抽桌固定種子完全可重現,批次統計與精確值在 5σ 內吻合且測試固定;重抽循環
誠實(永不保留零基礎手,無基礎牌組有明確空態);新匯入格式有夾具測試;範本載入
即顯示摘要;圖卡 PNG 與列印頁無任何卡圖;輪替分叉後黃金測試恆綠;`/ship-check` 通過。

## Phase 9 — 超越期:革命性功能(`08 §5C`,佢冇、而且好難有)

任務:
1. **P9.1 牌組數學體檢報告**:一鍵全套——重抽率、起手分級、關鍵組合(重抽條件化)、
   能量斷流、死手歸因 Top 3、獎賞風險——單頁報告 + 可分享圖卡。「60 秒體檢你副牌。」
   全部複用 Phase 2–7 組件,零新數學。
2. **P9.2 試抽桌 goldfish 模式**:逐回合抽牌(先攻/後攻、可選歷史規則開關),
   旁欄實時疊「到第 T 回合見到 X」精確曲線(`02 §6`)與接力事件檢查(`02 §6.5`);
   模擬與精確並排顯示。
3. **P9.3 訓練閉環**:試抽桌任意一手一鍵轉成訓練模式題目(「呢手出現概率係?」),
   接駁現有誤差紀錄。

DoD:體檢報告所有數字逐一可展開數學收據且與對應視圖一致;goldfish 每回合曲線值
與 `02 §6.2` 錨點吻合;訓練閉環題目揭曉值有黃金對應;`05 §F` 全項通過。
